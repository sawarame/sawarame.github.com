# 多言語対応方針 (i18n Policy)

本プロジェクトにおける多言語対応（日本語・英語）の実装ルールです。

## 1. 基本構成
*   **フレームワーク**: Docusaurus i18n を使用。
*   **デフォルト言語**: 日本語 (`ja`)。URLパス: `/`。
*   **追加言語**: 英語 (`en`)。URLパス: `/en/`。

## 2. 実装方法
*   **Reactコンポーネント内**: 
    *   `@docusaurus/Translate` コンポーネントを使用。
    *   静的な文字列には `translate()` 関数を使用。
    *   翻訳データは `i18n/en/code.json` に ID ベースで定義する。
*   **メタデータ**: 
    *   `i18n/en/docusaurus-theme-classic/navbar.json` などの設定ファイルを適切に更新する。
*   **ブログ・ドキュメント**: 
    *   `i18n/en/docusaurus-plugin-content-blog/` 等のディレクトリで個別にファイルを管理する。

## 3. 翻訳の優先順位
1.  ナビゲーション、フッターなどの共通UI。
2.  各ツールのタイトル、説明文、ボタンラベル。
3.  ブログ記事（主要なものから順次）。

## 4. エージェントへの指示
*   新しいコンポーネントを作成する際は、最初から ID を指定した `Translate` コンポーネントを使用し、将来的な多言語化を容易にすること。
*   英語翻訳を行う際は、技術用語の一貫性を保ち、簡潔で分かりやすい表現を選択すること。
