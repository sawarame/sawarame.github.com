# 設計原則 (Design Principles)

本プロジェクトにおけるツール開発の共通指針です。エージェントは新しいツールの作成や既存ツールの修正時、必ず以下の原則を遵守してください。

## 1. 完全ローカル処理 (Client-Side Only)
*   **原則**: ユーザーのデータ（画像、PDF、テキスト等）は、すべてブラウザ内（クライアントサイド）で処理し、外部サーバーへ送信してはならない。
*   **例外**: グローバルIPアドレスの取得など、外部APIの呼び出しが不可欠な場合に限る。その場合も、プライバシーに配慮し、不要なデータ送信は行わない。
*   **実装**: Canvas API, Web Worker, クライアントサイドライブラリ（jsQR, pdfjs-dist, JSZip等）を積極的に活用する。

## 2. 状態の永続化と共有
*   **URLパラメータ同期**: ツールの設定値（パスワードの長さ、QRの内容等）は、リアルタイムでURLのクエリパラメータに反映させる。これにより、設定状態をURLとして保存・共有可能にする。
*   **LocalStorage の活用**: 一時的なメモや設定など、ブラウザを閉じても保持すべきデータは `localStorage` を使用して永続化する。

## 3. UI/UX 規約
*   **アクションボタン**: 
    *   各出力結果には「コピー」ボタンを提供する。
    *   ダウンロード機能では、一貫したファイル名フォーマット（例: `YYYYMMDDHHMMSS`）を使用する。
    *   モバイル端末での操作性を考慮し、レスポンシブなレイアウト（ボタンのスタック等）を採用する。
*   **リアルタイムフィードバック**: 入力値の変更は即座にプレビューや結果に反映させる。
*   **クリーンなUI**: Docusaurus のテーマに馴染むデザインを心がけ、過度な装飾を避け、機能性を重視する。

## 4. セキュリティ
*   **クロール保護**: サーバーサイドAPIを呼び出す際は、Botやクローラーを検出し、必要に応じて制限を行う。
*   **サニタイズ**: ファイル名として使用される文字列から、システム予約文字（`/ \ : * ? " < > |`）を自動的に除去または置換する。
