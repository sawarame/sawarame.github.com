# 🤖 GEMINI.md - プロジェクト基本ルール（AI用）

このファイルは、AIエージェント（Gemini等）が本プロジェクトでのコーディングや操作を行う際に守るべき基本的なガイドラインです。
新しいタスクに着手する際は、常にこのルールを遵守してください。

## 1. プロジェクト概要
- **プロジェクト名**: sawara.me
- **種類**: ドキュメント・ブログサイト (ポートフォリオ等)
- **主要目的**: 美しく統一されたデザインと、管理しやすいページ・ドキュメント構造を提供する。

## 2. 技術スタック・環境
- **フレームワーク**: Docusaurus 
- **言語**: TypeScript / JavaScript, React
- **スタイリング**: カスタムCSS, CSS Modules (主に `src/pages/css/` 内などで管理)
- **パッケージマネージャー**: Yarn

## 3. コーディングの基本ルール
- **型安全**: 可能な限り TypeScript で型定義 `interface` / `type` を明記してください。`any` 型は極力避けてください。
- **コンポーネント設計**: React の Functional Component (FC) と Hooks を使用してください。
- **スタイリング**: 
  - 共通のスタイル（全体のベース設定など）や CSS Modules を活用し、整理されたディレクトリ構造に従って追加・修正してください。
  - Docusaurus が標準で提供しているクラスや仕組みは極力活かしてください。
  - レスポンシブデザインを意識して実装してください。

## 4. ディレクトリ構成と役割の理解
- `docs/` : ドキュメントページ用のマークダウンファイル群
- `blog/` : ブログ記事用のマークダウンファイル群
- `src/` : カスタムのReactコンポーネントやページ、CSSモジュール
- `static/` : 画像などの静的ファイル
- `docusaurus.config.ts` : サイト全体の設定ファイル

**作業時の注意:** 
「ページのデザイン変更」と「記事内容の修正」で参照すべき場所が異なります。目的に応じて正しいディレクトリを探すようにしてください。

## 5. コマンド・ワークフロー
開発用コマンドが必要な場合は、以下のコマンドを考慮してください。
- ローカルサーバーの起動: `yarn start`
- ビルドの実行: `yarn build`
- パッケージ追加: `yarn add <package>`

## 6. AIコーディング時の注意点 (AI向け指示)
- **不要な変更は加えない**: 指定されたタスク以外のファイルやコメントは勝手に削除・変更しないでください。
- **エラーチェック**: コードを生成・修正したあとは、文法エラーやTypeScriptの型エラーが出ないかを意識してください。
- **美しさとUXの重視**: ユーザーにとって魅力的なUIを設計するために、モダンで使いやすいデザインを意識してください。
- **コマンド実行の確認**: `rm` 等の非可逆の破壊的コマンドを行わないでください。ファイルの作成や読み書きは専用ツールを使用してください。
- **仕様書の更新維持**: ツール群に新しい機能を追加したり、既存の挙動や仕様を変更したりした場合は、必ず `SPEC.md` も併せて修正し、常に仕様書が実装と同期される状態を保ってください。
